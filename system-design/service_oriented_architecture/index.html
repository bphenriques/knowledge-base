    Service Oriented Architecture

    Referred in

    • Monolith
    • Microservices

    As opposed to Monolith, services do not share a database and all access must be done through a API exposed by the service. They may be in the same process (Monolith) or may be separated (Microservices). This reduces coupling.

    E.g.:

    • Orders -> RDBMS
    • Customers -> No SQL
    • Reservations -> Web Service
    • Menu -> RDBMS
